Patterns

    Do
  • Reserve for high priority, prominent messaging
  • Display one banner at a time whenever possible
  • Try to keep message length under 240 characters
  • Use the icon="" attribute only when the icon communicates something meaningful by itself.

    For example: If your banner message is asking a user to update a setting, it is appropriate to use the settings (settings) icon

    • Don't
  • Use banners to tell users about new features or other information that doesn't require the user to take action
  • (Generally) Use an icon if it's not something your users will recognize as a consequence of using your app(s)

For more in-depth design guidance, refer to Material.ioopen_in_new

How to use

Refer to the readmeopen_in_new for detailed information.


<myuw-banner
  message=""
  icon=""
  confirming-text=""
  confirming-url=""
  confirming-callback=""
  dismissive-text="">
</myuw-banner>

Attributes

  • message: The content of your banner message (keep it short and sweet
  • icon: (optional) A material icon to convey the context of your message
  • confirming-text: What should the "take action" button say?
  • dismissive-text: What should the "dismiss" button say?
  • confirming-url: URL to navigate to when "take action" button is clicked
  • confirming-callback: Name of function to execute when "take action" button is clicked (sets the button's onclick="" value)

Note: "confirming-url" and "confirming-callback" are mutually exclusive attributes. In cases where both are set, the component will set "confirming-url" and ignore "confirming-callback".

myuw-banner {
