Patterns

    Do
  • Set the app-url to the app's home page
  • CAUTIOUSLY utilize unused slots for task-related buttons/features

    For example: Your app doesn't have "notifications" functionality, but it does have an app settings menu. You could use the "myuw-notifications" slot to house that menu button

    • Don't
  • Populate slots with components you don't need
  • Utilize unused slots for navigation items
  • Put myuw-web-components in unintended slots

    For example: Don't put the myuw-notifications component in the "myuw-navigation" slot because you like it better on the left

For more in-depth design guidance, refer to Material.ioopen_in_new

How to use

Refer to the readmeopen_in_new for detailed information.


<myuw-app-bar
  theme-name=""
  app-name=""
  app-url="">
</myuw-app-bar>

Attributes

  • theme-name: Lightly bolded name of the "ecosystem" in which the app lives (e.g. MyUW, UMark, DoIT, etc.)
  • app-name: The name of the app
  • app-url: The URL to go to when the app name is clicked

Slots

The top app bar's slots are designed to house other myuw-web-components to ensure consistent placement and recognizable patterns wherever the bar is used.

  • myuw-navigation: Insert the myuw-drawer component
  • myuw-search: Insert the myuw-search component
  • myuw-notifications: Insert the myuw-notifications component
  • myuw-help: Insert the myuw-help component
  • myuw-profile: Insert the myuw-profile component
